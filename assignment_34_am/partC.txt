Q1: CONCEPTUAL

QUESTION:
K-Means is often called a 'greedy' algorithm. What does this mean? 
Can it get stuck in local minima? How does KMeans++ initialization help?

ANSWER:

Greedy Algorithm:
K-Means is called greedy because at each step it makes the locally optimal choice:
- Assigns each point to the nearest centroid
- Updates centroids based on current assignments

It does not consider future consequences, only minimizes current distance.

Local Minima:
Yes, K-Means can get stuck in local minima because:
- It depends heavily on initial centroid placement
- Different initializations can lead to different final clusters

KMeans++ Initialization:
KMeans++ improves initialization by:
- Choosing first centroid randomly
- Selecting next centroids far from existing ones

Benefits:
- Better spread of initial centroids
- Faster convergence
- Reduces chances of poor local minima

Q2: CODING

QUESTION:
Implement K-Means from scratch using NumPy.

CODE:

import numpy as np

def kmeans(X, k, max_iter=100):
    np.random.seed(42)

    # Step 1: Initialize centroids randomly
    indices = np.random.choice(len(X), k, replace=False)
    centroids = X[indices]

    for _ in range(max_iter):
        # Step 2: Assign clusters
        distances = np.linalg.norm(X[:, None] - centroids, axis=2)
        labels = np.argmin(distances, axis=1)

        # Step 3: Update centroids
        new_centroids = np.array([
            X[labels == i].mean(axis=0) if len(X[labels == i]) > 0 else centroids[i]
            for i in range(k)
        ])

        # Step 4: Check convergence
        if np.all(centroids == new_centroids):
            break

        centroids = new_centroids

    return labels, centroids

EXPLANATION:
- Computes distance of each point to all centroids
- Assigns closest centroid
- Updates centroids as mean of cluster points
- Stops when centroids stop changing

Q3: ANALYSIS

QUESTION:
K=5 gives silhouette score = 0.25. Is this good? What to do next?

ANSWER:

Silhouette Score Interpretation:
- Range: -1 to 1
- 1 → Perfect clustering
- 0 → Overlapping clusters
- <0 → Wrong clustering

Score = 0.25:
- Weak clustering
- Clusters are not well separated

What to investigate next:

1. Try different K values:
   - Use Elbow Method
   - Try K = 2, 3, 4

2. Check data distribution:
   - Use PCA or visualization
   - Look for natural groupings

3. Improve preprocessing:
   - Scaling
   - Remove noise/outliers

4. Try different algorithms:
   - DBSCAN
   - Agglomerative clustering

5. Use better initialization:
   - KMeans++

--------------------------------------------------

CONCLUSION:

- K-Means is simple but sensitive to initialization
- Evaluation metrics like silhouette help judge quality
- Always experiment with K and preprocessing
